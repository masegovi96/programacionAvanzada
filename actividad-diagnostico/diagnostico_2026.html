Actividad de Diagnóstico: Lista de Pendientes (To do List)

Objetivo

Evaluar tus habilidades en HTML semántico, CSS y JavaScript mediante la construcción de una aplicación funcional de gestión de tareas. No se trata de que sea perfecta, sino de ver cómo piensas y qué tanto dominas estas tecnologías.

¿Qué debes construir?

Una aplicación web de lista de pendientes (To do List) que funcione completamente en el navegador, sin backend ni base de datos externa.

Requisitos funcionales

Tu aplicación debe cumplir con los siguientes puntos. Implementa los que puedas; no es necesario terminarlos todos para entregar.

1
Agregar y eliminar tareas
El usuario debe poder escribir el texto de una tarea en un campo de entrada y agregarla a la lista al presionar un botón o la tecla Enter. Cada tarea debe tener un botón para eliminarla de la lista. Usa etiquetas HTML semánticas adecuadas (<ul>, <li>, <input>, <button>, etc.).
2
Marcar tareas como completadas
Al hacer clic sobre una tarea (o en un checkbox asociado), debe marcarse como completada. Las tareas completadas deben mostrarse visualmente distintas: al menos con texto tachado (text-decoration: line-through) y un color diferente (por ejemplo, gris). Volver a hacer clic la desmarca.
3
Filtrar tareas
Incluye tres botones o pestañas de filtro: Todas, Pendientes y Completadas. Al seleccionar un filtro, la lista debe mostrar únicamente las tareas correspondientes, sin recargar la página. El filtro activo debe tener una apariencia visual distinta.
4
Contador de tareas restantes
Muestra en algún lugar visible cuántas tareas quedan pendientes. Este número debe actualizarse automáticamente cada vez que se agrega, elimina o completa una tarea. Ejemplo: "3 tareas pendientes".
5
Persistencia con localStorage
Las tareas deben guardarse en localStorage del navegador para que, al recargar la página, no se pierdan. Cuando la app cargue, debe leer localStorage y restaurar la lista con el estado (pendiente/completada) de cada tarea.

Criterios de evaluación

Criterio Aspectos a observar
HTML semántico Uso apropiado de etiquetas (<ul>, <li>, <input>, <button>, atributos id / for, etc.).
CSS Estilos para tareas completadas (tachado, color), filtro activo resaltado y presentación general organizada. No es necesario que sea un diseño elaborado.
JavaScript Manipulación del DOM, manejo de eventos, lógica de filtrado y uso de localStorage.
Organización del código Legibilidad, separación de responsabilidades (HTML / CSS / JS), nombres de variables y funciones claros.

Entrega

  • Un único archivo .html con CSS y JS embebidos (en etiquetas <style> y <script>), o bien archivos separados enlazados correctamente.
  • Tiempo estimado: 60 minutos.

Subir la actividad a GitHub

Una vez que hayas terminado tu actividad, sigue estos pasos para subirla a tu repositorio:

  1. Crea (o clona) tu repositorio del curso en GitHub con la estructura solicitada en Classroom.
  2. Dentro del repositorio, crea una carpeta llamada exactamente Actividad diagnóstico.
  3. Coloca dentro de esa carpeta todos los archivos relacionados con tu actividad (HTML, CSS, JS, imágenes, etc.).
  4. Realiza un commit con un mensaje descriptivo, por ejemplo: feat: actividad diagnóstico TODO List.
  5. Haz push a tu rama principal y verifica en GitHub que los archivos se visualicen correctamente.
  6. Entrega el enlace a tu repositorio a través de Classroom antes de que concluya el tiempo.
Importante: Se revisarán los logs de Git para validar la hora exacta de la entrega. Asegúrate de hacer el push dentro del tiempo asignado.
