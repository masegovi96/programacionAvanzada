async / await

¿Qué es async / await?

async y await son dos palabras clave introducidas en ES2017 que permiten trabajar con Promises de forma más legible. No reemplazan a las Promises; son una capa encima de ellas que evita el encadenamiento de .then() y produce código que se lee de manera secuencial, igual que si fuera síncrono.

Usando fetch() con .then() el código crece hacia la derecha (callbacks anidados). Con async/await el mismo código crece hacia abajo, que es la dirección natural de lectura.

Regla de oro: await solo puede usarse dentro de una función marcada con async. Una función async siempre devuelve una Promise, aunque no haya ningún await dentro.

La palabra clave async

Colocar async antes de la declaración de una función la convierte en una función asíncrona. Su comportamiento cambia en dos aspectos:

  • Siempre devuelve una Promise, incluso si el valor de retorno es un dato primitivo.
  • Habilita el uso de await dentro de su cuerpo.
// Función normal — devuelve el número 42
function obtenerValor() {
  return 42;
}

// Función async — devuelve Promise<42>
async function obtenerValor() {
  return 42;
}

La palabra clave await

await se coloca delante de una expresión que devuelve una Promise. Le indica a JavaScript que pause la ejecución de la función asíncrona en ese punto hasta que la Promise se resuelva, y luego extrae el valor resultante. El resto del programa (fuera de la función) sigue ejecutándose con normalidad.

Importante: await no bloquea el hilo principal de JavaScript. Solo pausa la función async en la que está; el motor de JavaScript continúa ejecutando cualquier otro código mientras espera la respuesta.

async / await vs .then()

Los dos fragmentos de código siguientes hacen exactamente lo mismo: consultan un post de JSONPlaceholder e imprimen su título. La diferencia es solo de legibilidad.

Con .then() encadenado

fetch('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => {
    if (!response.ok) throw new Error(response.status);
    return response.json();
  })
  .then(data => {
    console.log(data.title);
  })
  .catch(error => {
    console.error('Error:', error);
  });

Con async / await

async function obtenerPost() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/posts/1');

    if (!response.ok) throw new Error(response.status);

    const data = await response.json();
    console.log(data.title);

  } catch (error) {
    console.error('Error:', error);
  }
}

obtenerPost();
Con async/await el manejo de errores se realiza con el bloque estándar try / catch, igual que en código síncrono, en lugar de encadenar un .catch().

Ejemplos de código

GET — Leer un recurso

async function obtenerUsuario(id) {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);

    if (!response.ok) throw new Error('Usuario no encontrado');

    const usuario = await response.json();
    console.log(`Nombre: ${usuario.name} — Email: ${usuario.email}`);

  } catch (error) {
    console.error(error.message);
  }
}

obtenerUsuario(1);

POST — Crear un recurso

async function crearPost(titulo, contenido) {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/posts', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ title: titulo, body: contenido, userId: 1 })
    });

    const nuevoPost = await response.json();
    console.log('Creado con id:', nuevoPost.id);

  } catch (error) {
    console.error('Error al crear:', error);
  }
}

crearPost('Mi título', 'El contenido del post');

Múltiples peticiones en paralelo con Promise.all()

Cuando necesitas hacer varias peticiones independientes y esperar a que todas terminen, usar await en secuencia es ineficiente porque espera una por una. Promise.all() las lanza todas a la vez y espera a que todas se resuelvan. 

async function obtenerDatos() {
  try {
    // Se lanzan las dos peticiones al mismo tiempo
    const [resUsuarios, resPosts] = await Promise.all([
      fetch('https://jsonplaceholder.typicode.com/users'),
      fetch('https://jsonplaceholder.typicode.com/posts')
    ]);

    const usuarios = await resUsuarios.json();
    const posts    = await resPosts.json();

    console.log(`${usuarios.length} usuarios y ${posts.length} posts cargados`);

  } catch (error) {
    console.error('Una petición falló:', error);
  }
}

obtenerDatos();

Resumen

Concepto Descripción
async function Declara una función asíncrona que siempre devuelve una Promise.
await Pausa la función asíncrona hasta que la Promise se resuelva y extrae su valor.
try / catch Reemplaza al .catch(); captura errores de red y errores lanzados manualmente.
response.ok Propiedad booleana que indica si el código de estado está entre 200 y 299.
response.json() Parsea el cuerpo de la respuesta como JSON; también es una Promise, por eso necesita await.
Promise.all() Ejecuta múltiples Promises en paralelo y espera a que todas se resuelvan.
