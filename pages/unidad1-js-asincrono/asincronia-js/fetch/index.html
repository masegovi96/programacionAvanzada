fetch()

¿Qué es fetch()?

fetch() es una función integrada en los navegadores modernos que permite realizar peticiones HTTP de forma asíncrona, es decir, sin bloquear la ejecución del resto del programa mientras se espera la respuesta del servidor.

Antes de fetch(), la forma estándar de hacer peticiones asíncronas era usando XMLHttpRequest (XHR), una API más compleja y verbosa. fetch() llegó para simplificar ese proceso con una sintaxis más limpia basada en Promesas (Promises).

Clave: fetch() devuelve una Promise. Eso significa que no devuelve la respuesta de inmediato; en cambio, devuelve un objeto que eventualmente se resolverá con la respuesta del servidor (o se rechazará si hubo un error de red).

¿Qué es una Promise?

Una Promise (promesa) representa el resultado eventual de una operación asíncrona. Puede estar en uno de tres estados:

Pending

La operación aún está en curso. Ni cumplida ni rechazada.

Fulfilled

La operación terminó con éxito. El valor prometido está disponible.

Rejected

La operación falló. Se dispara el manejador de error.

Para reaccionar a una Promise se encadenan los métodos .then() y .catch(). El primero recibe el valor resuelto; el segundo captura cualquier error.

Firma y parámetros de fetch()

fetch(url, options)
Parámetro Tipo Descripción
url string La URL del endpoint al que se hace la petición. Obligatorio.
options object Objeto de configuración opcional. Permite definir método, cabeceras, cuerpo, etc.

Opciones más utilizadas

{
  method:  'GET',            // GET | POST | PUT | PATCH | DELETE
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ nombre: 'valor' })
  // body solo se usa con POST, PUT y PATCH
}

¿Cómo funciona el flujo?

Cuando se llama a fetch(), el navegador envía la petición HTTP en segundo plano y devuelve inmediatamente una Promise. El código siguiente al fetch() se ejecuta sin esperar. Cuando el servidor responde, se activa el callback del .then().

Importante: fetch() solo rechaza la Promise si hay un error de red (sin conexión, DNS no resuelto, etc.). Una respuesta 404 o 500 del servidor no lanza un error automáticamente: la Promise se resuelve igual. Por eso hay que revisar la propiedad response.ok manualmente.

La respuesta que llega en el primer .then() es un objeto Response. Para leer los datos como JSON hay que llamar a response.json(), que también devuelve una Promise, razón por la que se encadena un segundo .then().

Ejemplos de código

GET — Obtener datos

// Petición GET a la API pública JSONPlaceholder
fetch('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => {
    // Verificar que la respuesta fue exitosa
    if (!response.ok) {
      throw new Error('Error en la petición: ' + response.status);
    }
    // Convertir la respuesta a JSON (también es una Promise)
    return response.json();
  })
  .then(data => {
    // Aquí ya tenemos el objeto JavaScript con los datos
    console.log(data.title);
  })
  .catch(error => {
    // Solo se activa si hay error de red
    console.error('Error de red:', error);
  });

POST — Enviar datos

// Petición POST: crear un nuevo recurso
fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    title:  'Mi post',
    body:   'Contenido del post',
    userId: 1
  })
})
  .then(response => response.json())
  .then(data => {
    console.log('Recurso creado con id:', data.id);
  })
  .catch(error => {
    console.error('Error:', error);
  });

DELETE — Eliminar un recurso

// Petición DELETE: eliminar el post con id=1
fetch('https://jsonplaceholder.typicode.com/posts/1', {
  method: 'DELETE'
})
  .then(response => {
    if (response.ok) {
      console.log('Recurso eliminado correctamente');
    }
  })
  .catch(error => {
    console.error('Error:', error);
  });

Resumen del flujo

Paso Qué ocurre Método / Propiedad
1 Se lanza la petición HTTP fetch(url, options)
2 Se recibe la respuesta del servidor (metadatos) .then(response => ...)
3 Se verifica que la respuesta sea exitosa response.ok · response.status
4 Se parsea el cuerpo como JSON response.json()
5 Se trabaja con los datos obtenidos .then(data => ...)
6 Se captura cualquier error de red .catch(error => ...)
