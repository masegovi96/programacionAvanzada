Introducción a las APIs

¿Qué es una API?

API son las siglas de Application Programming Interface (Interfaz de Programación de Aplicaciones). En términos simples, es un contrato de comunicación entre dos sistemas de software: define qué operaciones están disponibles, cómo pedirlas y en qué formato se recibirá la respuesta.

Piensa en una API como una ventanilla de atención: no necesitas saber cómo funciona el sistema interno; solo necesitas saber qué le puedes pedir y qué te va a devolver.

Definición formal: Una API es un conjunto de reglas y definiciones que permite que una aplicación se comunique con otra, abstrayendo los detalles de implementación y exponiendo únicamente la funcionalidad necesaria.

Interoperabilidad

Permite que aplicaciones escritas en diferentes lenguajes y plataformas se comuniquen entre sí.

Abstracción

El cliente no necesita conocer los detalles internos del servidor; solo la interfaz pública.

Reutilización

Una misma API puede ser consumida por una app web, una app móvil y un servicio de terceros a la vez.

Separación de capas

Desacopla el frontend del backend, permitiendo que cada uno evolucione de forma independiente.

La analogía del restaurante

Una de las formas más sencillas de entender una API es imaginar un restaurante. Cada parte del proceso tiene su equivalente en el desarrollo web:

Cliente

El Frontend: quien hace la solicitud.

Mesero

La API: lleva la orden y regresa la respuesta.

Cocina

El Backend: procesa la solicitud y prepara los datos.

Restaurante Equivalente en software Descripción
Cliente Frontend (navegador / app) Quien genera la solicitud: un clic en un botón, cargar una página, etc.
Mesero API Recibe la petición del cliente, la lleva a la cocina y devuelve el resultado. No cocina, solo comunica.
Cocina Backend (servidor / base de datos) Procesa la lógica, consulta la base de datos y prepara los datos que serán devueltos.
Menú Documentación de la API Lista de lo que puedes pedir: rutas disponibles, parámetros aceptados y formato de respuesta.
El mesero no necesita saber cómo se prepara cada platillo, y el cliente no necesita entrar a la cocina. Esa separación de responsabilidades es exactamente lo que hace poderosa a una API.

Una API pública para practicar

Existe una API de acceso libre llamada JSONPlaceholder que simula un servidor REST con datos de prueba. Es perfecta para aprender porque no requiere registro, no requiere token de autenticación y responde de inmediato.

JSONPlaceholder

Fake REST API for testing and prototyping.

Visitar jsonplaceholder.typicode.com

Esta API expone varios recursos de prueba:

  • /posts — 100 publicaciones de blog
  • /comments — comentarios asociados a publicaciones
  • /albums — álbumes de fotos
  • /photos — 5 000 fotos
  • /todos — tareas pendientes
  • /users — 10 usuarios con perfiles completos

Estructura de una respuesta de API

La mayoría de las APIs modernas devuelven los datos en formato JSON (JavaScript Object Notation). JSON es un formato de texto ligero y legible tanto por humanos como por máquinas.

Cuando haces una petición a https://jsonplaceholder.typicode.com/posts/1, el servidor responde con un objeto JSON que describe una publicación.

Respuesta: un objeto (recurso único)

Al solicitar un recurso específico por su id, la API devuelve un objeto JSON:

// GET https://jsonplaceholder.typicode.com/posts/1
{
  "userId": 1,
  "id":     1,
  "title":  "sunt aut facere repellat provident occaecati excepturi optio reprehenderit",
  "body":   "quia et suscipit\nsuscipit recusandae consequuntur..."
}

Respuesta: un arreglo (colección de recursos)

Al solicitar una colección sin filtros, la API devuelve un arreglo de objetos JSON:

// GET https://jsonplaceholder.typicode.com/posts
[
  {
    "userId": 1,
    "id":     1,
    "title":  "sunt aut facere repellat...",
    "body":   "quia et suscipit..."
  },
  {
    "userId": 1,
    "id":     2,
    "title":  "qui est esse",
    "body":   "est rerum tempore vitae..."
  },
  // ... 98 objetos más
]

Tipos de datos en JSON

JSON soporta seis tipos de valores. Aquí un ejemplo de un usuario con todos ellos:

// GET https://jsonplaceholder.typicode.com/users/1
{
  "id":       1,                          // número
  "name":    "Leanne Graham",           // cadena de texto
  "active":  true,                      // booleano
  "score":   null,                      // nulo
  "address": {                       // objeto anidado
    "street": "Kulas Light",
    "city":   "Gwenborough",
    "zipcode":"92998-3874",
    "geo": {
      "lat": "-37.3159",
      "lng": "81.1496"
    }
  },
  "tags": ["admin", "user"]           // arreglo
}

¿Qué es un endpoint?

Un endpoint (punto de acceso) es una URL específica dentro de una API a la que puedes enviar una petición para realizar una operación concreta. Cada endpoint combina tres elementos:

URL base

La dirección raíz del servidor. Ej.: https://jsonplaceholder.typicode.com

Ruta (path)

El recurso específico que se quiere operar. Ej.: /posts, /users/3

Método HTTP

La operación a realizar: GET, POST, PUT, DELETE.

Mismo recurso, distintas operaciones. La ruta /posts/1 puede comportarse diferente según el método HTTP que uses: GET obtiene el post, PUT lo reemplaza, DELETE lo elimina.

Métodos HTTP más comunes (CRUD)

Método Operación CRUD Descripción
GET Read (Leer) Obtiene datos del servidor. No modifica nada.
POST Create (Crear) Envía datos al servidor para crear un nuevo recurso.
PUT Update (Actualizar) Reemplaza completamente un recurso existente.
DELETE Delete (Eliminar) Elimina un recurso del servidor.

Endpoints de JSONPlaceholder

Método Endpoint Descripción Ejemplo de respuesta
GET /posts Obtiene todos los posts Arreglo de 100 objetos
GET /posts/1 Obtiene el post con id=1 Un objeto con userId, id, title, body
GET /posts/1/comments Obtiene los comentarios del post 1 Arreglo de objetos comentario
GET /users Obtiene todos los usuarios Arreglo de 10 objetos usuario
POST /posts Crea un nuevo post (simulado) El objeto creado con id: 101
PUT /posts/1 Reemplaza el post con id=1 El objeto actualizado
DELETE /posts/1 Elimina el post con id=1 Objeto vacío {}

Anatomía de un endpoint

  https://jsonplaceholder.typicode.com/posts/1?_limit=5
  ──────────────────────────────────── ───── ─ ─────────
           │                            │    │     │
        URL base                      ruta  id  query string
                                                (parámetros opcionales)
  • URL base: dirección del servidor que ofrece la API.
  • Ruta: el recurso que se solicita (/posts, /users, etc.).
  • Parámetro de ruta: identifica un recurso concreto dentro de la colección (el /1).
  • Query string: parámetros opcionales para filtrar, ordenar o paginar resultados (?_limit=5).
